# INC-2025-030 — Wrong order totals after pricing change

## Summary
Orders computed incorrect totals for 12 minutes. No 5xx spike; the defect was
silent and caught by a customer report.

## Root cause
A commit changed the pricing formula to `total = price * (qty - 1)`, an
off-by-one logic bug, and returned a 500 on quantity zero. A code defect
(see RB-03).

## Resolution
Because the same commit carried an unrelated fix, the team shipped a forward
patch with a regression test rather than reverting. CI green, then deploy.
Followed RB-03; action was "patch" with human approval.

## Duration
12m. SLA impact ≈ €1800 at €150/min (correctness, not availability).
