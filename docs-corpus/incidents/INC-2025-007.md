# INC-2025-007 — Boot failure after dependency bump

## Summary
The service failed to boot for 3 minutes after a deploy; the container
crash-looped.

## Root cause
A lockfile change upgraded a library with a breaking API change. The import
failed at startup. A dependency regression (see RB-04).

## Resolution
Reverted the lockfile to the pinned last-known-good version. The service booted
clean on redeploy. Followed RB-04.

## Duration
3m 20s. SLA impact ≈ €500 at €150/min.
